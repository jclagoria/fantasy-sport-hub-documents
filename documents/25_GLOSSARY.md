## üìö Domain Terms (Fantasy Sports)

**Commissioner**: User with special permissions to manage a league, resolve disputes, and modify settings.

**Draft**: Initial process where users select players for their teams before the season begins.

**Lineup**: Set of active players on a team for a specific week/match.

**Roster**: Complete list of players on a team, including starters and substitutes.

**Waiver**: System for acquiring free agents with rotating priority.

**Trade**: Transaction where two teams exchange players.

**Playoff**: Elimination phase at the end of the regular season.

**Scoring**: System that converts sports statistics into fantasy points.

**Live Scoring**: Real-time score updates during matches.

**Bye Week**: Week when a team doesn't play (common in NFL).

**Bench**: Players on the roster who are not active for that week.

## üîß Technical Terms

### Architecture

**Plugin Architecture**: Design pattern where functionality is added through interchangeable components.

**Event Sourcing**: Pattern where state changes are stored as a sequence of events.

**CQRS**: Command Query Responsibility Segregation - separation of read and write models.

**ACL (Anti-Corruption Layer)**: Layer that translates between external and internal models.

**Circuit Breaker**: Pattern that prevents calls to failed services.

**Aggregate**: Cluster of domain objects treated as a unit.

**Read Model**: Projection optimized for queries (CQRS).

**Write Model**: Model optimized for commands/changes (CQRS).

### Reactive Programming

**Backpressure**: Mechanism that controls data flow between producer and consumer.

**Flux**: Reactive stream of 0-N elements (Project Reactor).

**Mono**: Reactive stream of 0-1 elements (Project Reactor).

**Publisher**: Data source in reactive streams.

**Subscriber**: Data consumer in reactive streams.

**Operator**: Function that transforms reactive streams.
